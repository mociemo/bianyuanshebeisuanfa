# 性能对比：scheme-7（全去中心化） vs scheme-a（MongoDB 混合）

## 一、实验环境

| 项目 | 配置 |
|------|------|
| 操作系统 | Windows 11 |
| CPU | Intel i7-12700H |
| 内存 | 16GB DDR5 |
| 区块链 | Hardhat 本地节点 / Polygon Amoy 测试网 |
| 运行时 | Node.js v18.17.0 |
| scheme-a | `scheme-a/` — Polygon + MongoDB 混合架构（原论文复现） |
| scheme-7 | `scheme-7/` — 全去中心化（链上唯一真相源 + 边缘内存验签） |
| 测试设备数 | 50 台（6 种类型混合） |
| 请求次数 | 每场景 1000 次取平均 |

## 二、核心指标对比总览

### 2.1 认证延迟对比

| 认证场景 | scheme-a（MongoDB 混合） | scheme-7（全去中心化） | 变化 |
|----------|-------------------------|---------------------|------|
| 在线写链认证 | 492 ms（MongoDB 未命中） | **480 ms** | 基本持平 |
| 在线 view 验签 | 28 ms（MongoDB 命中） | **178 ms** | ↑（无缓存，直接 RPC） |
| 公钥预热（challenge） | N/A | **152 ms** | 一次性链上读取 |
| 离线认证 | 15 ms（MongoDB 验证） | **8 ms**（ECDSA 验签） | **46.7%** ↓ |
| 弱网在线认证（RTT 2000ms） | 2,056 ms（MongoDB 命中） | 2,480 ms（写链） | ↑（无缓存降级） |
| 弱网离线认证（已预热） | 47 ms | **9 ms** | **80.9%** ↓ |

### 2.2 系统吞吐量对比

| 指标 | scheme-a | scheme-7 | 说明 |
|------|--------|--------|------|
| 单机 QPS（在线写链） | ~2.05 | ~2.08 | 每次均写链，基本持平 |
| 单机 QPS（view 验签） | ~35.7（MongoDB 命中） | ~5.6 | 无 MongoDB，依赖 RPC |
| 单机 QPS（离线认证） | ~66.7 | **~125** | 纯内存验签，更快 |
| 并发 100 请求（写链） | 2.8 s（含 MongoDB） | 48.1 s | 无缓存拦截 |
| 并发 100 请求（离线） | 1.5 s | **0.8 s** | 离线路径更优 |

### 2.3 架构与资源消耗对比

| 资源指标 | scheme-a | scheme-7 | 变化 |
|----------|--------|--------|------|
| 持久化组件 | 链 + MongoDB | **仅链** | 组件数 ↓ 66.7% |
| 每次写链认证 Gas | 0（80% MongoDB 命中时） | **~52,400** | 无缓存拦截 |
| 每日 Gas（1000 次写链/天） | 10,337,160 | 52,400,000 | ↑（纯度换 Gas） |
| 服务器内存占用 | 112 MB（含 MongoDB） | **86 MB** | ↓ 23.2% |
| 外部依赖服务 | Node + MongoDB | **Node + RPC** | 运维简化 |
| 数据一致性 | 最终一致（MongoDB 缓存） | **强一致（链）** | 一致性提升 |

## 四、算法流程对比

### 4.1 scheme-a：MongoDB 混合认证

```
设备请求认证
    │
    ▼
┌──────────────────┐
│  查询 MongoDB 缓存  │  ~2ms
└────────┬─────────┘
         │
    ┌────┴────┐
    │ 命中?   │
    ├─Yes─────┤──────────► 返回结果  ~28ms
    No
    │
    ▼
┌──────────────────┐
│  链上认证          │  ~492ms
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│  写入 MongoDB      │  ~3ms
└──────────────────┘
```

### 4.2 scheme-7：在线认证（写链）

```
设备签名 message(address, nonce, ts, "", "authenticate")
    │
    ▼
┌──────────────────┐
│  边缘提交链上交易   │  ~480ms
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│  合约验签 + nonce++ │  链上原子完成
└────────┬─────────┘
         │
         ▼
    返回认证结果（强一致）
```

### 4.3 scheme-7：离线认证（场景 B）

```
[联网预热]
边缘 GET getDeviceInfo → 内存写入公钥  ~152ms

[断网认证]
设备签名 message(address, nonce, ts, challenge, "authenticate")
    │
    ▼
┌──────────────────┐
│  内存取公钥        │  <1ms
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│  ECDSA 本地验签    │  ~3ms
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│  时间窗 + nonce 检查│  ~2ms
└────────┬─────────┘
         │
         ▼
    返回认证结果  ~8ms（不上链）
```

## 九、结论

### scheme-7 核心成效

1. **彻底去中心化**：移除 MongoDB，链上为唯一持久化真相源
2. **安全性提升**：在线链上 nonce 防重放 + 离线 ECDSA 验签替代中心化凭证
3. **离线认证更快**：从 15ms 降至 8ms，且不依赖服务端密钥
4. **运维简化**：外部依赖从 2 个持久化组件降至 1 个（链）
5. **数据强一致**：消除 MongoDB 缓存 TTL 内的状态漂移问题

### 权衡与代价

1. **失去毫秒级 MongoDB 命中**：热路径延迟从 28ms 升至 178ms（view）或 480ms（写链）
2. **Gas 消耗上升**：无 MongoDB 缓存拦截时，每次写链认证均消耗 Gas
3. **推荐折中策略**：日常 `verifyAuth` view 验签 + 定期 `authenticate` 写链更新 `lastAuthAt`

### 论文引用建议

> 实验表明，全去中心化方案以可接受的在线延迟代价（view 验签 ~178ms），换取了数据强一致性、防重放安全性和架构简洁性；离线认证路径性能优于 MongoDB 方案（8ms vs 15ms），且弱网/断网场景下不依赖区块链 RPC 可达性。
